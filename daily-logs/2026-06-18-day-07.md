# Day 07 — 2026-06-18

## Topic
Linux file search tools — which, find, locate (HackTheBox module)

## Resources used
- HackTheBox Academy — Linux Fundamentals module

## Tasks completed
- Learned `which` — locates the executable path of a program (e.g. `which python`)
- Learned `find` syntax and filter options:
  - `-type f` — restrict to files only
  - `-name *.conf` — match by filename pattern
  - `-user root` — filter by file owner
  - `-size +20k` — filter by file size
  - `-newermt 2020-03-03` — filter by modification date
  - `-exec ls -al {} \;` — run a command on each result
  - `2>/dev/null` — suppress error output
- Learned `locate` — searches a pre-built database (faster than `find`, fewer filters)
- Learned `sudo updatedb` — refreshes the locate database before searching
- Compared when to use `find` (precise, filtered search) vs `locate` (fast, broad search)

## Tools used
- HackTheBox Academy interactive terminal

## Key takeaway
`find` with date and owner filters (`-user root -newermt <date>`) is a real incident response technique — it's how analysts hunt for recently modified or suspiciously-owned config files on a system that might be compromised. `which` quickly confirms what tools are available on a host, useful both for attackers checking their toolkit and defenders auditing what's installed.
